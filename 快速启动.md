# 🚀 快速启动指南

## 第一次使用

1. **安装依赖**（如果还没安装）
   - 双击运行 `快速部署.bat`（会自动安装依赖）
   - 或手动运行：`npm install`

2. **配置 API 密钥**
   - 创建 `.env.local` 文件
   - 添加：`OPENAI_API_KEY=你的API密钥`

3. **启动服务器**
   - 双击 `启动服务器.bat`
   - 或运行：`npm run dev`

4. **访问网站**
   - 打开浏览器
   - 访问：http://localhost:3000

## 日常使用

### 每次使用前

**重启电脑后，需要重新启动服务器！**

1. 双击 `启动服务器.bat`
2. 等待看到 "Ready" 消息
3. 访问 http://localhost:3000

### 如果网页打不开

1. **最简单的方法**：双击 `重启服务器.bat`
   - 会自动停止冲突进程
   - 清理缓存
   - 重新启动服务器

2. **手动检查**
   - 查看终端窗口是否显示错误
   - 检查端口是否被占用
   - 查看 `TROUBLESHOOTING.md` 获取详细帮助

## 📁 脚本说明

| 脚本文件 | 用途 | 使用场景 |
|---------|------|---------|
| `启动服务器.bat` | 启动开发服务器 | 每次需要访问网站时 |
| `重启服务器.bat` | 重启服务器（解决冲突） | 网页打不开时 |
| `快速部署.bat` | Git 初始化和部署准备 | 第一次部署到 Vercel |

## 💡 小贴士

- **保持服务器运行**：启动后不要关闭终端窗口，关闭窗口服务器会停止
- **多个项目**：如果同时开发多个项目，可能会端口冲突，使用 `重启服务器.bat` 解决
- **查看日志**：运行服务器的窗口会显示所有请求和错误信息

## ❓ 常见问题

**Q: 为什么重启电脑后网站打不开？**  
A: 开发服务器不会自动启动，需要手动运行 `启动服务器.bat`

**Q: 如何停止服务器？**  
A: 在运行服务器的窗口按 `Ctrl + C`，或直接关闭窗口

**Q: 可以最小化窗口吗？**  
A: 可以，服务器会在后台继续运行

**Q: 网页显示错误怎么办？**  
A: 查看终端窗口的错误信息，或查看 `TROUBLESHOOTING.md`
