# 如何运行修复脚本

## 🚀 最简单的方法：双击运行

### 步骤：

1. **打开文件管理器（Windows 资源管理器）**
   - 按 `Win + E` 打开
   - 或从开始菜单打开

2. **导航到项目目录**
   - 地址栏输入：`E:\LTNS`
   - 或手动导航到该文件夹

3. **找到文件**
   - 找到 `立即修复Vercel部署.bat` 文件
   - 文件名显示为：**立即修复Vercel部署.bat**

4. **双击运行**
   - 双击该文件
   - 会自动打开一个黑色窗口（CMD）
   - 显示执行过程

5. **等待完成**
   - 脚本会自动：
     - 添加文件到 Git
     - 提交更改
     - 推送到 GitHub
   - 完成后窗口会显示 "按任意键继续..."
   - 按任意键关闭窗口

---

## 📋 方法 2：在 CMD 中运行

1. **打开 CMD（命令提示符）**
   - 按 `Win + R`
   - 输入 `cmd`
   - 按回车

2. **切换到项目目录**
   ```cmd
   cd /d E:\LTNS
   ```

3. **运行脚本**
   ```cmd
   立即修复Vercel部署.bat
   ```

---

## 📋 方法 3：在 PowerShell 中运行

1. **打开 PowerShell**
   - 在项目文件夹中，按住 `Shift` + 右键
   - 选择 "在此处打开 PowerShell 窗口"

2. **运行脚本**
   ```powershell
   cmd /c 立即修复Vercel部署.bat
   ```
   或
   ```powershell
   .\立即修复Vercel部署.bat
   ```

---

## ⚠️ 如果遇到问题

### 问题 1：提示"找不到文件"
- 确保在 `E:\LTNS` 目录下
- 检查文件名是否正确（包括扩展名 `.bat`）

### 问题 2：提示"Git 命令找不到"
- 需要先配置 Git，运行：`配置Git.bat`

### 问题 3：提示"推送失败"
- 检查网络连接
- 确认 GitHub 账户权限
- 可能需要输入 GitHub 用户名和密码
